文章点评

致力于开源项目可以为你的简历增权

原文链接：HOW CONTRIBUTING TO OPEN SOURCE PROJECTS CAN IMPROVE YOUR ENGINEERING RESUME

适兕点评：

在编程的生产活动中存在着大量的隐性知识，那么人才的培养和识别就势必面临着很多的问题，那么如果有那么一批开发者，在公开的GitHub等开源共同体聚集之所不断的展示着自己高超的技能和卓越的技巧，那么即使这群人有着特别的价值观，只要能够利用到其一技之长，是很多有识之士的不二选择。本文则是从另外的角度来阐述，告诉年轻的开发者，开源可以帮助你展示自己的才华！

破除开源软件不需商业的神话

原文链接：Dissecting the Myth that Open Source software is not commercial

适兕点评：

来自Karl Fogel 在2016年的文章，在现实中适兕也会遇到很多人关于开源软件的纯洁道德高度，貌似完全的利他主义，不食人间烟火的味道。是的，我们无法阻止别人如何理解，甚至连辩解都有些困难，因为社会的多元性，秉承一种价值观的人，对于软件二进制授权模式就是认为是最完美的形式，有些人则秉承，只要不是强制，就不会付费！这里是愚昧、社会主义、缺乏教育、人性的自私等等诸多因素的合成。

开源和健康领袖合力抵抗冠状病毒

原文链接：Open source and health leaders join forces to fight coronavirus

适兕点评：

民主的优势就在于，当发生诸如全球性灾难这样的事情时，每个个体都会根据自己的知识去尝试做些什么，反之，则是等待指示。Linux 基金会作为开源软件的一个组织，以强大的号召力得到大量开发者的信任和商业公司的青睐。这个组织没有去等待，而是主动出击，成立了新的子基金会：

Linux Foundation Public Health (LFPH) builds, secures, and sustains open source software to help public health authorities (PHAs) combat COVID-19 and future epidemics.

开源共同体呼吁 GitHub “放弃ICE”

原文链接：The Open Source Community Is Calling on Github to ‘Drop ICE'

适兕点评：

作为一名当下的中华人民共和国居民，我非常难以理解这等做法，政府做事，一个不是实体组织的虚拟组织，可以发动大家签名，去抵抗政府已经颁布的法令。简直不敢思议。开源是世界的，全人类的事情，不可以有掺杂其它的东西。那么开源的政治化色彩和工具，也就越来越浓了。不过这只是开源的又一个副作用罢了，了解了开源之后，一个人很难不使用这样的思考方式。

2020年以及未来的开源学习

原文链接：Open-Source Learning for 2020 and Beyond

适兕点评：

John Dewey 和实用主义，开源的哲学指导，尤其是在学习方面。Middlebury 果然已经走在了前列。“that education is the fundamental method of social progress and reform” 以建设共同体的方式在互联网上构建相应的学习组织，或许是个不错的方式，但是做这件事之前，需要做更多的准备工作：接受监管、获得资格、融资、生源……开源大学失败的例子很多，就在于共同体的建设，而不是某项技术本身。

Cloudera 和开源的问题

原文链接：Cloudera and the Problem With Open Source

适兕点评：

在探索开源的奥秘过程中，适兕越发的感受到愿意购买开源的哪种心智上的变化的难得可贵。经济也是一种文化现象，就中国的历史而言，很多成为商品和服务的内容都是通过教育而实现的：城市化、抽水马桶、摩托车……购买软件，厂商提供源代码，这是天经地义的事情！然而却被垄断知识的邪恶厂商所控制了！然而此去经年，这样的方式竟然被大众所接受了，提供源代码反而成了不正常的事情。还得靠布道师们去启蒙！嗯，这就是历史，这也是人类自己干的事情。

OASIS 提供基金会即服务

原文链接：Open Source Communities Embrace New Option for Running Independent Foundations

适兕点评：

伴随着开源的崛起，开源非营利基金会也进入了其黄金发展期，连中国国内的很多人都看到了这一点，于是关于基金会的建设也一直有人在尝试去搞，那么这个东西，自顶向下的方式有个什么问题了呢？不是金钱收入的直接来源的开发者和工程师哪里找？所谓的社会资本谁来做担保？项目凭什么将所有权收走？这个完全不符合人类的本能。

红帽和 GraalVM 项目创建了“Mandrel” GraalVM 分发版

原文链接：Red Hat and GraalVM Community Create “Mandrel” GraalVM Distro

适兕点评：

JDK 面临四分五裂的局面，AWS、Alibaba等众多公有云厂商，分别根据自身的情况提供对应的JDK 开源+商业版，在这个时候，所有的公司都在争夺，想象当年的Unix发行版之战，BSD 发行版之战也是同样的道理。Edge分离出Chromium，和Chrome是亦敌亦友的关系。OpenJDK 能走多远？始终是个大问题。

云中的开源

原文链接：Open source in the clouds

适兕点评：

SkySQL， MariaDB 的云计算版，即数据库即服务！至少我现在看不出来其有何商业意义，或许是从中国本土的情况出发的，MariaDB创始人谈到：Open source is not just about being open or less expensive. You can’t have something open if it’s not understandable. 可是将科技视为巫术的人，怎么会去想着理解来龙去脉了呢？

开源在美国政府的四大神话

原文链接：4 myths about open source in government

适兕点评：

即使是在开源的诞生地美国，RedHat 也是花了大量的精力来做一些思想和心智上的宣传和输出的，尤其是实用主义方面的内容。比如，针对美国政府的游说，这篇文章就是最佳的例证。对比起来，中国本土，由于威权的关系，开源还不能成为代表纳税人的采购名单，总而言之，这里面的问题就复杂的多。 California Code 那么中国的那个省市会推出自己的代码托管平台了呢？

RedHat 尝试将Kafka 带进 Kubernetes

原文链接：Red Hat Platform Brings Kafka Closer to Kubernetes

适兕点评：

随着技术栈的升级，开源的项目由于选择的增多而整体的复杂性升高。这要从Linux到OpenStack再到Kubernetes逐步的看起来，这对于老牌的如RedHat来说，既是挑战，也是机会。比如它对OpenStack的支持，再比如其在接近应用的合作伙伴选择，Operator Hub 能否战胜？JBoss 是成功还是失败？

Apache Cassandra 共同体是如何停止内部争斗的

原文链接：How the Apache Cassandra community stopped fighting to build its best release yet

适兕点评：

一个开源项目，其背后的开发主力——共同体 ，的建设是至为关键的，某种程度上，可以说是项目成败的最重要的部分，例外的是商业公司主导的开源项目，但失去了共同体的活力，其实和传统的商业公司雇佣员工也没有太大区别了，甚至反而会更加的累，投入更加的多，毕竟要去解释很多事情。Apache Cassandra 内部在4.0开发之前发生了一系列的事情：“I’m an absolute sucker for process and quality improvement and Cassandra 4.0 has this in spades。 这属于正常，Flink 和Blink 的故事也值得写写。
